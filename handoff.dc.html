핵심 원칙 · CORE RULES
이 규칙들은 특정 샘플 데이터가 아니라 모든 KR/US 내부자·주요주주 공시 데이터에 공통 적용된다. 새로운 issuer/ticker/reporting owner가 들어와도 동일한 렌더링 규칙을 따른다. KR metadata가 비어 있거나 불완전한 경우 issuer 및 filing date는 본문 표지/발행회사 표/접수번호/제목에서 fallback 추출하고, 모든 fallback 값은 source_mapping에 출처를 기록한다.
  • KR D002 · KR 거래계획보고서 · US SEC Form 3·4·5(3/A·5/A 포함)를 하나의 공통 UI·스키마로 표현한다.
  • 거래(transactions)와 보유(holdings)는 분리한다. transactionCode와 취득/처분(A/D)도 별도 필드로 분리한다.
  • 가격 0과 null은 다르게 처리한다(0=정상값, null=미기재). 수량은 decimal 허용, 반올림 금지.
  • footnote / 비고는 의미 해석의 핵심 정보다. 값은 추정하지 말고 없으면 null + status로 표기한다.
  • 범위(v6.3): 본 공통 UI는 KR D002 + US Form 3/4/5(3·4·5/A 포함)만 다룬다. KR 거래계획보고서(00633)는 범위 제외(out-of-scope) — 예정 거래 공시로서 D002의 보유·변동 및 Form 3/4/5의 발생 거래와 성격이 다르므로 planned transaction 관련 UI/schema(planned_transactions·current/expected_holdings·plan_scale_assessment·trading_plan·Planned/Trading plan badge)는 사용하지 않는다. 입력 시 status=out_of_scope로 처리하고 D002로 오인하지 않는다.
  • 화면 변경 시 스키마 · 매핑 · Empty State · 추출 프롬프트를 함께 갱신한다.
  • UI는 다크 프리미엄 금융 SaaS. 한국 관례 색 취득·증가=Red / 처분·감소=Blue / 중립·미기재=Gray.
Supported Inputs · Extraction Pipeline (B-1)
PDF 또는 DART 원문은 LLM에 직접 넣지 않는다. 먼저 DART parser로 section tree · tables · text blocks · metadata를 추출하고, 그 결과를 common schema extraction prompt에 투입한다.
  • KR D002: PDF/HTML → DART parser → metadata.json · contents.json · tables.json · tables_mdkv.md(+ qa verdict.json · qa_report) → 프롬프트 투입. PDF 바이너리 직접 입력 금지.
  • US Form 3/4/5: SEC .txt/.xml/.html 원문에서 reportingOwner · issuer · nonDerivativeTable · derivativeTable · footnotes · ownerSignature 추출 → 프롬프트 투입.
  • Source Mapping: 파서 산출물이 있으면 source_mapping.raw_form · parser_notes[] · parser_qa{verdict, section_pass_count, section_total_count}에 출처/신뢰도 기록. 추출 신뢰도와 UI 데이터는 분리.
Quick Copy · 공통 스키마 추출 프롬프트
원본 DART D002 / SEC Form 3·4·5를 common JSON schema로 변환하는 LLM 프롬프트.
You are a financial-disclosure normalizer. Convert ONE insider-ownership
filing — Korean DART "D002" / KR "거래계획보고서" OR US SEC "Form 3 / 4 / 5 /
3-A / 5-A" — into the JSON schema "insider-ownership-common-v1" (Appendix A).
The input may arrive as XML, HTML, rendered table text, or pipe-delimited text.

OUTPUT
- Return ONLY one valid JSON object matching the schema. No prose, no markdown.
- Keep every schema key. Use null for any value not present in the source.
- NEVER guess or infer unknown values. If unsure, use null and add a short
  note to source_mapping.parser_notes[].

DOCUMENT META
- Detect country/source: DART→KR, SEC→US. Set form_type exactly
  (D002 | 3 | 4 | 5 | 3/A | 5/A). is_amendment=true when form_type ends with /A.
- report_type: KR→신규|변동|정정 ; US→initial(3)|change(4)|annual(5)|amendment(/A).
- For amendments, extract date_of_original_submission. filing_date = filed date.
- Issuer identity: issuer.cik (US) / company_code (KR corp_code) is the STABLE
  identity. Same company may file under different tickers over time (e.g.
  issuerTradingSymbol "FB" in 2022, "META" in 2026, same issuerCik 0001326801).
  Store ticker as the document's value AND display_ticker; preserve prior tickers
  in former_symbols[] and prior names in former_names[] (from SEC header FORMER
  COMPANY). Set grouping_key = "issuer_cik:" (US) or "corp_code:" (KR).
  Grouping/dedup MUST use cik/corp_code, never ticker alone.

SEPARATION RULES (critical)
- Put real movements in transactions[]; static positions in holdings[]. A US
  filing may have BOTH nonDerivativeTransaction AND nonDerivativeHolding — keep
  them in their own arrays. Never merge.
- Keep transaction_code and acquired_disposed_code (A/D) as SEPARATE fields.
  Store code="S", a/d from transactionAcquiredDisposedCode — do not collapse.
- Derivatives / rights (RSU, Stock Option, 신주인수권, 전환사채권,
  신주인수권부사채권, 교환사채권) go to derivative_or_rights[], with
  underlying_security_title / shares / exercise price / exercise_date /
  expiration_date.

KR D002 SPECIFICS
- 직전보고서→holding_summary.previous, 이번보고서→current, 증감→change. No prior
  report → previous=null, status="no_previous_report".
- Separate quantity change from ratio change (opposite signs possible when a
  rights exercise issues new shares: qty up, ratio down). Judge
  quantity_change_direction / ratio_change_direction independently
  (increase|decrease|neutral).
- 합계/소계 row in 세부변동내역 → is_summary_row=true (exclude from per-row math).
- 거래계획 보고일자 → trading_plan.kr_plan_report_date.
- "주식배당(+)" → transaction_category="stock_dividend", a/d="A", price=null
  (price_missing_reason="not_applicable_or_not_reported"). Not a market buy.
- KR 거래방법/보고사유: preserve the original raw label in transaction_reason_raw
  (e.g. "장내매수(+)"), AND store transaction_category ("market_purchase") and
  transaction_category_label ("장내매수"). Never store only the +/- stripped label.
- Keep total_specific_securities (특정증권등) and shares_only (주권) as SEPARATE
  objects, even if equal.
- 종류별 소유내역 table: A,B,C…J may be COLUMN codes, not security titles. Respect
  original table header/colspan/rowspan; do not treat a lone A~J cell as a heading.

US FORM SPECIFICS
- noSecuritiesOwned=1 → holding_summary.no_securities_owned=true,
  status="no_securities_owned", holdings=[], missing_tables_allowed=true.
  noSecuritiesOwned=0 is NOT "no securities" (Form 3 holding-only).
- reportingOwner may NOT be an individual officer. owner_type from
  individual | institution | financial_institution | company | unknown. Derive
  relationship from is_director/is_officer/is_ten_percent_owner/is_other; all
  false → "관계 미기재". Do NOT hardcode "officer". A reporting owner may hold
  MULTIPLE roles at once (Director + Officer + 10% Owner) — populate
  roles_display[] with ALL of them (UI shows every role as a chip, never a single
  collapsed text). Same for KR multi-role.
- officer_title: store SEC officerTitle verbatim in officer_title_raw AND
  officer_title_display (no restoration/cleanup even if truncated like "EVP,
  Chief Human Resources Off").
- directOrIndirectOwnership: D→"direct", I→"indirect". Copy natureOfOwnership
  verbatim (By Trust, By spouse, 401(k), By LLC ...). 401(k)/retirement/benefit
  plan → ownership_subtype="retirement_plan_account".
- aff10b5One=1 → trading_plan.is_10b5_1=true. Extract adoption_date from footnote
  ("adopted on 11/10/2025" → "2025-11-10").
- Form 5: preserve form5_flags {not_subject_to_section16, form3_holdings_reported,
  form4_transactions_reported} (null if absent). periodOfReport is the report base
  date, NOT a transaction date — keep separate; transactions may predate it by
  years (contains_historical=true, transaction_date_range{from,to}). Not an error.
- transactionTimeliness "L" → transaction_timeliness="L", is_late_reported=true.
  SEPARATE from transaction_code — never treat "L" as a code.

CLASSIFICATION (conservative — never from code alone)
- Combine code + a/d + price + footnote text + nature_of_ownership + KR
  reason/note + table context. When unsure → "unknown".
- G + price 0 + NO footnote → gift_or_transfer,
  classification_confidence="low_without_footnote". Do NOT invent GRAT/trust.
- G + footnote(GRAT, annuity payment, trust, trustee, annuitant, spouse, "mere
  change in form of beneficial ownership", "exempt from Section 16", "Rule
  16a-13") → transaction_category="trust_transfer_or_annuity_payment",
  transaction_subtype="GRAT_annuity_payment". Keep code="G".
- A + price 0 + footnote(RSU/award/grant/no consideration/vesting) →
  compensation_award / RSU_award (same category as KR 자사주상여금). A + price>0 +
  no footnote → acquisition_or_purchase. A + plan/employee purchase →
  plan_purchase.
- W: keep code="W". footnote(inheritance/will/estate/"previously omitted") →
  inheritance_or_estate_transfer. Do not coerce to buy/sell.

M / EXERCISE / DERIVATIVE LINKING
- transactionCode "M" (or securityTitle/footnote with Exercise, Vesting, Option,
  Stock Option, RSU, Restricted Stock Unit, derivativeTransaction,
  underlyingSecurity, conversionOrExercisePrice) →
  transaction_category="exercise_or_vesting". Do NOT label a non-derivative M
  (price 0/A) as a plain free acquisition.
- ALWAYS parse derivativeTable (derivativeTransaction AND derivativeHolding) into
  derivative_or_rights[] with id, underlying_security_title,
  underlying_security_shares, conversion_or_exercise_price, exercise_date,
  expiration_date. Link the non-derivative M row to its derivative via
  transactions[].linked_derivative_ids = [derivative id]. This applies to Form 5
  too — never skip derivativeTable on a Form 5. Empty derivativeTable → empty state.

PAIRED A/D TRANSFER (Form 4 & 5)
- Same date + security + shares + footnote as a D (out of trust) and an A (to
  reporting person): KEEP BOTH rows untouched. ADD a UI-only group
  transaction_group_id, group_display_type:"paired_transfer",
  paired_transfer:{from,to,shares,security_title,footnote_ids}.

NUMBERS & PRICES
- Quantities DECIMAL-allowed (35.944, 21836.814, 6606.917). Preserve exactly.
- price 0/0.00 (Gift, award, exercise) is VALID: number 0, price_is_zero=true,
  price_zero_valid=true, price_zero_reason. Missing price → null,
  price_missing_reason="not_reported". NEVER merge 0 and null.

FOOTNOTES
- Extract every footnote with id (F1, F2, …) and full text into footnotes[].
- Row-level footnote_ids[] AND field_footnotes {transaction_shares,
  transaction_price_per_share, shares_owned_following_transaction,
  security_title, nature_of_ownership} — a footnote may attach to a single field.
- Stock Units in holdings: securityTitle with Stock Unit(s)/GSU/RSU/PSU →
  holding_type="stock_unit", compensation_security=true, link
  vesting_schedule_footnote_ids. Keep in holdings[] (do not move).
- remarks is a top-level string, separate from footnotes[]. EX-24 / EX-24.POA
  (Power of Attorney) → exhibits[]{type, description, is_transaction_data:false}.
  NEVER parse exhibits as transaction/holding rows.

KR 거래계획보고서 — OUT OF SCOPE
- KR 임원ㆍ주요주주 특정증권등 "거래계획보고서" (form_subtype_code=="00633",
  title contains "거래계획보고서", or item_list contains 거래계획/거래규모 판단/
  거래계획 완료후 예상 소유상황) is OUT OF SCOPE for this common UI. It is a
  PLANNED (future) transaction filing, distinct from D002 ownership/holdings and
  US Form 3/4/5 executed-transaction reports.
- Do NOT coerce it into D002. Set form_type="KR_TRADING_PLAN",
  document_meta.status="out_of_scope" and STOP (do not populate transactions/
  holdings). No planned_transactions / current_holdings_before_plan /
  expected_holdings_after_plan / plan_scale_assessment / trading_plan in this scope.

KR FALLBACK / SOURCE PRIORITY (metadata may be empty)
- KR DART metadata.corps[] (corp_name, symbols, corp_code) and filing_date may be
  BLANK. Do NOT leave issuer null. Use the body "발행회사에 관한 사항" table as
  source of truth: issuer.name, ticker/company_code, market (법인구분: 코스닥상장법인
  →KOSDAQ, 유가증권→KOSPI), shares_outstanding (발행주식 총수). Record
  issuer.source_priority="body_table_issuer_info" and log it in source_mapping.
- KR filing/event dates fallback order: (1) cover/표지 text (거래시작일,
  보고의무발생일, 보고서작성기준일), (2) title date, (3) receipt_number first 8
  digits (20260609→2026-06-09), (4) metadata date, (5) null. Keep them SEPARATE:
  보고의무발생일→event_date, 보고서작성기준일→report_base_date, 거래시작일→
  planned_trade_start_date. Do NOT mistake 거래시작일 for filing_date. Record each
  in filing_dates.date_source_priority and source_mapping.

EMPTY STATE HINTS
- Fill ui_empty_states.{transactions|holdings|derivative_or_rights|footnotes}
  with {is_empty, reason, message} when empty, else null. Examples:
  reason:"form3_initial_no_transactions", "no_securities_owned",
  "no_derivative_or_rights", "no_footnotes".

- If a parser QA artifact (verdict.json) accompanies the source, record it in
  source_mapping.parser_qa {verdict, section_pass_count, section_total_count,
  notes}. It is reliability metadata only, separate from UI data; null if absent.

Return the JSON now.

2 · Current UI Rules

현재 구현 기준(최종). 누적 버전 이력은 Appendix D 참조.

List / Search

  • Multi-transaction filing(v8.9): 같은 filing 내 F/S/G transaction은 개별 row로 분리(단일 net 합산 금지). 메인 요약은 「거래 N건 · S 3건·G 1건」 형태. 예: Stevens Jun = S 200k/100k/400k + G 307.5k(모두 By Trust). Filing Detail·거래목록 상세는 개별 row 표시.
  • 10b5-1은 transaction/footnote 단위: filing 전체 일괄 적용 금지. 같은 filing 안에서도 F=비10b5 / S=10b5 가능(예: Puri·Robertson·Kress). row별 is10b5로 판단, footnote 연결. Adoption date: Shah 2024-11-25, Dabiri 2025-12-10, Kress 2025-03-04 등 footnote/remarks에 보존.
  • Trust / indirect ownership: directOrIndirectOwnership=I + natureOfOwnership(By Trust / Envy Trust / Horne Teter Family Living Trust 등) + footnote 연결 필수. Filing Detail 보유 내역에 Trust 명의 노출.
  • Weighted average / price range: 단일=price, 가중평균=Avg 표시, range는 footnote에 가격범위(예: Shah $175.82~$176.57, Kress $171.85~$177.92). 내부 value_source: reported|calculated|unavailable 구분 유지.
  • NVDA 배치 표기 정정(v8.8): US NVDA 12건 = 12 filing / 1 issuer group(cik 0001045810) / 12 reporting owner(12 issuer/13 issuer group 아님). 벡트 D002 포함 시 13 filing / 2 issuer group / 13 reporting-owner filing.
  • Event cluster 규칙(v8.8): 동일 issuer + 동일 transaction_date + 동일 transactionCode + 유사 reason이 여러 reporting owner에서 동시 발생(예: NVDA 2026-03-18 F 6명) → issuer card 내부에서 cluster summary("F · 세금원천징수 · 6명 동시 보고") + 펼침 시 개별 owner row 노출, row별 「거래목록 보기」 유지.
  • 10b5-1 vs KR 거래계획 분리: KR 거래계획보고서·planned_transactions 등은 out-of-scope. US Rule 10b5-1은 in-scope(Form 4 footnote/status badge로 유지, Planned/Trading plan badge로 바꾸지 않음). adoption date는 footnote/remarks에 보존.
  • UI placeholder 금지: 사용자-facing UI에 (qty)·undefined·NaN·raw null 노출 금지. 추출 실패 시 「수량 미추출」/「단가 미추출」/「확인 필요」 같은 사용자 문구로 표시, schema 값은 null 유지.
  • Parser Input Pipeline 유지: PDF/HTML은 LLM에 직접 투입 금지. DART parser로 metadata/contents/tables/tables_mdkv 추출 후 prompt에 투입. KR D002는 파서 산출물 함께 업로드 권장.
  • 그룹형 단일 뷰(테이블·카드 뷰 제거, 보기 전환 UI 없음). 종목 기준 그룹 고정(group_by=issuer/ticker).
  • 필터 toolbar(v8.7): 단일 행 compact toolbar — [국가 segmented] · [변동유형 segmented] · [공시유형 select dropdown]. 컨테이너 배경 매우 약하게, 비선택 투명, 선택만 강조. 공시유형은 길이 줄이려 dropdown으로 전환. 정렬은 필터에서 제외하고 결과 안내 줄 우측에 배치(라벨 없음, 값만 노출). placeholder 「회사명, 티커, 내부자·보고자 검색」.
  • 회사명 primary, ticker/code secondary. 검색바 아래는 KPI 카드 대신 한 줄 메타.
  • 그룹 헤더 CTA(v7.3): 「최신 공시 보기 ›」 1개만 — issuer group의 최신 filing 1건 상세로 이동. 「종목 거래 목록 보기」 제거. Form 4 고정 아님 — filing_date 최신 기준, 동일 날짜는 거래/변동 있음 > Form4 > Form5 > Form3 > 정정 > holding-only/no-securities tie-break(Late·Amendment·No securities 비숨김). KR은 최신 D002 기준. 헤더 전체 클릭 없음.
  • 그룹 헤더 identity: [국가] 회사명 ticker·market 한 줄(국가 badge 좌, 회사명 primary, ticker·market secondary 우) + 아래 줄 공시·보고자 수·최신 공시일. 사용자-facing 문구에 7섹션·공통 스키마 등 내부 용어 미사용.
